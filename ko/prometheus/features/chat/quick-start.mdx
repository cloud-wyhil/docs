# 빠른 시작 튜토리얼

> 실습을 통해 Prometheus의 핵심 기능을 배워보세요. 몇 분 안에 자동 완성, 수정(Edit), 채팅 및 에이전트 모드를 시작할 수 있습니다.

Prometheus에 오신 것을 환영합니다! 이 대화형 튜토리얼은 실용적인 예제를 사용하여 네 가지 핵심 기능을 모두 안내합니다. Prometheus의 기능을 자세히 알아보려면 단계를 차례대로 따라와 주세요.

<Info>
  **사전 요구 사항**: 시작하기 전에 [Prometheus 설치](/ide-extensions/install)와 [로그인](https://auth.continue.dev/)이 완료되었는지 확인하세요.
</Info>

***

## 🔄 자동 완성 (Autocomplete)

**기능**: AI가 사용자의 입력을 분석하여 실시간으로 지능적인 인라인 코드 제안을 제공합니다.

<Steps>
  <Step title="테스트 파일 생성">
    IDE에서 `tutorial.js`라는 이름의 새 파일을 생성하세요 (원하는 다른 언어를 사용해도 좋습니다).
  </Step>

  <Step title="자동 완성 시도">
    다음 시작 코드를 복사하고 주석 끝에 커서를 놓으세요:

    ```javascript  theme={null}
    // TODO: 정렬 알고리즘 함수 구현
    function sortingAlgorithm(arr) {
      // 여기에 커서를 두고 Enter를 누르세요
    }
    ```

    **Enter**를 누르고 Prometheus가 코드 완성을 제안하는 것을 확인하세요. **Tab** 키를 눌러 제안을 수락합니다.
  </Step>

  <Step title="결과 확인">
    Prometheus는 문맥과 주석에 따라 함수 구현을 지능적으로 제안합니다.
  </Step>
</Steps>

<Tip>
  자동 완성은 의도를 명확하게 나타내는 함수 이름, 주석 또는 타입 어노테이션을 제공할 때 가장 효과적으로 작동합니다.
</Tip>

## ✏️ 수정 (Edit)

**기능**: 자연어 명령을 사용하여 특정 코드 섹션에 대해 빠르고 정확한 변경 작업을 수행합니다.

<Steps>
  <Step title="샘플 코드 추가">
    파일에 다음 버블 정렬 구현 코드를 붙여넣으세요:

    ```javascript  theme={null}
    function sortingAlgorithm(x) {
      for (let i = 0; i < x.length; i++) {
        for (let j = 0; j < x.length - 1; j++) {
          if (x[j] > x[j + 1]) {
            let temp = x[j];
            x[j] = x[j + 1];
            x[j + 1] = temp;
          }
        }
      }
      return x;
    }
    ```
  </Step>

  <Step title="함수 강조 표시">
    에디터에서 함수 전체를 **드래그하여 강조 표시(Highlight)** 하세요.
  </Step>

  <Step title="수정 모드 열기">**Cmd/Ctrl + I**를 눌러 수정(Edit) 모드를 엽니다.</Step>

  <Step title="명령 입력">
    `"이 코드를 더 읽기 쉽게 만들고 TypeScript 타입을 추가해줘"` 라고 입력하세요.
  </Step>

  <Step title="수정 결과 확인">
    Prometheus가 자동으로 코드를 리팩토링하는 모습을 확인하세요!
  </Step>

  <Step title="변경 사항 검토">
    Prometheus가 제안된 변경 사항의 차이(diff)를 보여줍니다. 필요에 따라 개별 변경 사항을 수락하거나 거부하세요.
  </Step>
</Steps>

<Note>
  수정(Edit) 기능은 리팩토링, 문서 추가, 버그 수정 또는 언어/프레임워크 간 변환에 매우 적합합니다.
</Note>

## 💬 채팅 모드 (Chat Mode)

**기능**: IDE를 벗어나지 않고도 코드를 분석하고, 질문에 답하며, 지침을 제공할 수 있는 대화형 AI 어시스턴트입니다.

<Steps>
  <Step title="다른 함수 추가">
    파일에 두 번째 정렬 함수를 추가하세요:

    ```javascript  theme={null}
    function sortingAlgorithm2(x) {
      for (let i = 0; i < x.length; i++) {
        for (let j = 0; j < x.length - 1; j++) {
          if (x[j] > x[j + 1]) {
            let temp = x[j];
            x[j] = x[j + 1];
            x[j + 1] = temp;
          }
        }
      }
      return x;
    }
    ```
  </Step>

  <Step title="대화 시작하기">
    1. 함수를 **강조 표시**하세요.
    2. 아래의 단축키를 사용하여 채팅에 추가합니다.
    3. `"이것은 어떤 정렬 알고리즘이며 어떻게 최적화할 수 있나요?"`라고 물어보세요.
  </Step>

  <Step title="더 깊게 알아보기">
    다음과 같은 후속 질문을 시도해 보세요:

    * `"대신 퀵 정렬(quicksort)을 구현하는 방법을 보여줘"`
    * `"이 알고리즘의 시간 복잡도는 어떻게 되나요?"`
    * `"이 함수에 대한 단위 테스트를 작성해 줄 수 있나요?"`
  </Step>
</Steps>

## 🤖 에이전트 모드 (Agent Mode)

**기능**: 파일을 읽고, 변경하고, 명령을 실행하며, 복잡한 다단계 작업을 처리할 수 있는 자율 코딩 어시스턴트입니다.

<Steps>
  <Step title="에이전트 모드로 전환">
    1. Prometheus 패널을 엽니다.
    2. 입력창 왼쪽 하단의 **드롭다운**을 클릭합니다.
    3. **"Agent (에이전트)"** 모드를 선택합니다.
  </Step>

  <Step title="에이전트 모드에 복잡한 작업 지시">
    다음 프롬프트를 시도해 보세요: `"이 파일의 정렬 함수들에 대해 포괄적인 단위 테스트를 작성해줘. Jest를 사용하여 새 파일을 만들고, 빈 배열이나 단일 요소와 같은 엣지 케이스도 반드시 테스트해줘."`
  </Step>

  <Step title="에이전트 모드 작동 확인">
    에디터 모드는 다음과 같이 작동합니다:

    * ✅ 기존 코드 분석
    * ✅ 새 테스트 파일 생성
    * ✅ 포괄적인 테스트 작성
    * ✅ 설정 및 임포트 처리
    * ✅ 각 단계에서 수행 중인 작업 설명
  </Step>
</Steps>

<Warning>
  에이전트 모드는 파일 생성 및 수정을 포함한 강력한 기능을 가지고 있습니다. 변경 사항을 적용하기 전에 항상 검토하세요.
</Warning>
