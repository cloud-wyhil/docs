# Prometheus 계획 모드 – 안전한 읽기 전용 코드 탐색

> 변경 사항을 적용하기 전에 읽기 전용 도구, 검색 및 분석을 사용하여 코드베이스를 안전하게 탐색하고 이해하기 위해 Prometheus에서 계획 모드를 사용하는 방법을 알아봅니다.

## 계획(Plan) 모드란 무엇인가요?

계획 모드는 코드베이스에 대한 읽기 전용 액세스 권한을 제공하는 제한된 환경입니다. 어떠한 수정도 가하지 않고 코드를 이해하고, 변경 사항을 계획하며 안전하게 탐색할 수 있도록 설계되었습니다.

### 계획 모드의 주요 기능은 무엇인가요?

* **읽기 전용 도구**: 위험 없이 파일 액세스, 검색 및 분석을 수행합니다.
* **안전한 탐색**: 낯선 코드베이스를 파악하는 데 적합합니다.
* **계획 수립에 집중**: 실행 전에 구현 전략을 개발합니다.
* **MCP 지원**: 내장된 읽기 전용 도구와 함께 모든 MCP 도구를 사용할 수 있습니다.

### 계획 모드 작동 방식

계획 모드는 사용 가능한 도구를 읽기 전용 작업으로만 제한합니다. 즉, 다음과 같은 작업이 가능합니다:

* 프로젝트의 모든 파일 읽기
* grep 및 glob 패턴을 사용하여 코드 검색
* 저장소 구조 및 변경 사항(diff) 보기
* 추가 컨텍스트를 위해 웹 콘텐츠 가져오기
* 모든 MCP 도구 사용

하지만 다음과 같은 작업은 불가능합니다:

* 파일 생성, 편집 또는 삭제
* 터미널 명령 실행
* 시스템 변경 수행

도구 및 사용법에 대한 자세한 내용은 에이전트 및 계획 모드를 모두 다루는 [에이전트 설명서](/ide-extensions/agent/how-it-works)를 참조하세요.

### 계획 모드의 일반적인 워크플로우는 무엇인가요?

1. 탐색 및 이해를 위해 **계획 모드에서 시작**합니다.
2. 모델의 도움을 받아 **접근 방식을 개발**합니다.
3. 구현할 준비가 되면 **에이전트 모드로 전환**합니다.

<Tip>
  계획 모드는 채팅 및 에이전트 모드와 동일한 인터페이스 및 컨텍스트 기능을 공유합니다. 다른 모드와 마찬가지로 `@` 컨텍스트 제공자를 사용하거나 코드를 강조 표시할 수 있습니다.
</Tip>
