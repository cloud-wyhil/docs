description: 'Mintlify는 docs.json 파일을 통해 문서의 디자인과 느낌을 완벽하게 제어할 수 있게 해줍니다'
icon: 'gear'
---

모든 Mintlify 사이트에는 핵심 구성 설정이 포함된 `docs.json` 파일이 필요합니다. 아래에서 각 [속성(Properties)](#properties)에 대해 자세히 알아보세요.

## 속성 (Properties)

<ResponseField name="name" type="string" required>
프로젝트 이름입니다. 글로벌 제목으로 사용됩니다.

예: `mintlify`

</ResponseField>

<ResponseField name="navigation" type="Navigation[]" required>
  그룹 내의 모든 페이지를 포함하는 그룹 배열입니다.
  <Expandable title="내비게이션 (Navigation)">
    <ResponseField name="group" type="string">
    그룹의 이름입니다.

    예: `Settings`

    </ResponseField>
    <ResponseField name="pages" type="string[]">
    페이지 역할을 할 마크다운 파일들의 상대 경로입니다.

    예: `["customization", "page"]`

    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="logo" type="string or object">
  로고 이미지의 경로 또는 "라이트" 및 "다크" 모드 로고 이미지 경로가 포함된 객체입니다.
  <Expandable title="로고 (Logo)">
    <ResponseField name="light" type="string">
      라이트 모드에서의 로고 경로
    </ResponseField>
    <ResponseField name="dark" type="string">
      다크 모드에서의 로고 경로
    </ResponseField>
    <ResponseField name="href" type="string" default="/">
      로고를 클릭했을 때 연결되는 링크
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="favicon" type="string">
  파비콘(Favicon) 이미지 경로
</ResponseField>

<ResponseField name="colors" type="Colors">
  글로벌 테마를 위한 헥사(Hex) 색상 코드
  <Expandable title="색상 (Colors)">
    <ResponseField name="primary" type="string" required>
      주요 색상입니다. 라이트 모드에서 강조된 콘텐츠, 섹션 헤더, 악센트 등에 가장 자주 사용됩니다.
    </ResponseField>
    <ResponseField name="light" type="string">
      다크 모드를 위한 주요 색상입니다. 다크 모드에서 강조된 콘텐츠, 섹션 헤더, 악센트 등에 가장 자주 사용됩니다.
    </ResponseField>
    <ResponseField name="dark" type="string">
      중요한 버튼을 위한 주요 색상입니다.
    </ResponseField>
    <ResponseField name="background" type="object">
      라이트 및 다크 모드 모두에서의 배경 색상
      <Expandable title="객체 (Object)">
        <ResponseField name="light" type="string" required>
          라이트 모드 배경의 헥사 색상 코드
        </ResponseField>
        <ResponseField name="dark" type="string" required>
          다크 모드 배경의 헥사 색상 코드
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="topbarLinks" type="TopbarLink[]">
  상단 바(Topbar)에 포함할 링크의 `name`과 `url` 배열
  <Expandable title="상단 바 링크 (TopbarLink)">
    <ResponseField name="name" type="string">
    버튼의 이름입니다.

    예: `Contact us`
    </ResponseField>
    <ResponseField name="url" type="string">
    버튼을 클릭했을 때의 URL입니다. 예: `https://mintlify.com/docs`
    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="topbarCtaButton" type="Call to Action">
  <Expandable title="상단 바 CTA 버튼 (Call to Action)">
    <ResponseField name="type" type={'"link" or "github"'} default="link">
    `link`는 버튼을 보여줍니다. `github`는 GitHub 스타 수를 포함하여 제공된 URL의 리포지토리 정보를 보여줍니다.
    </ResponseField>
    <ResponseField name="url" type="string">
    `link`인 경우: 버튼이 연결되는 링크입니다.
    
    `github`인 경우: GitHub 정보를 불러올 리포지토리 링크입니다.
    </ResponseField>
    <ResponseField name="name" type="string">
    버튼 내부의 텍스트입니다. `type`이 `link`인 경우에만 필수입니다.
    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="versions" type="string[]">
  버전 이름 배열입니다. 내비게이션 바의 드롭다운을 통해 서로 다른 버전의 문서를 보여주길 원하는 경우에만 사용하세요.
</ResponseField>

<ResponseField name="anchors" type="Anchor[]">
  `icon`, `color`, `url`을 포함하는 앵커(Anchor) 배열입니다.
  <Expandable title="앵커 (Anchor)">
    <ResponseField name="icon" type="string">
    앵커의 특징을 나타내는 [Font Awesome](https://fontawesome.com/search?q=heart) 아이콘입니다.

    예: `comments`
    </ResponseField>
    <ResponseField name="name" type="string">
    앵커 라벨의 이름입니다.

    예: `Community`
    </ResponseField>
    <ResponseField name="url" type="string">
      해당 앵커에 어떤 페이지가 들어갈지를 결정하는 URL의 시작 부분입니다. 일반적으로 페이지를 넣은 폴더의 이름입니다.
    </ResponseField>
    <ResponseField name="color" type="string">
      앵커 아이콘 배경의 헥사 색상입니다. 각각 헥사 색상을 값으로 가지는 `from`과 `to` 속성이 포함된 객체를 전달하여 그라데이션으로 설정할 수도 있습니다.
    </ResponseField>
    <ResponseField name="version" type="string">
      올바른 문서 버전이 선택될 때까지 앵커를 숨기고 싶을 때 사용합니다.
    </ResponseField>
    <ResponseField name="isDefaultHidden" type="boolean" default="false">
      누군가 직접 내부 문서로 링크를 보내기 전까지 앵커를 숨기려면 `true`를 전달하세요.
    </ResponseField>
    <ResponseField name="iconType" default="duotone" type="string">
      "brands", "duotone", "light", "sharp-solid", "solid", "thin" 중 하나
    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="topAnchor" type="Object">
  최상단 앵커에 대한 기본 구성을 덮어씁니다.
  <Expandable title="객체 (Object)">
    <ResponseField name="name" default="Documentation" type="string">
      최상단 앵커의 이름
    </ResponseField>
    <ResponseField name="icon" default="book-open" type="string">
      Font Awesome 아이콘.
    </ResponseField>
    <ResponseField name="iconType" default="duotone" type="string">
      "brands", "duotone", "light", "sharp-solid", "solid", "thin" 중 하나
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="tabs" type="Tabs[]">
  내비게이션 탭 배열입니다.
  <Expandable title="탭 (Tabs)">
    <ResponseField name="name" type="string">
      탭 라벨의 이름입니다.
    </ResponseField>
    <ResponseField name="url" type="string">
      해당 탭에 어떤 페이지가 들어갈지를 결정하는 URL의 시작 부분입니다. 일반적으로 페이지를 넣은 폴더의 이름입니다.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="api" type="API">
  API 설정을 위한 구성입니다. API 페이지에 대한 자세한 내용은 [API 컴포넌트](/api-playground/demo)에서 확인하세요.
  <Expandable title="API">
    <ResponseField name="baseUrl" type="string">
      모든 API 엔드포인트의 기본 URL입니다. `baseUrl`이 배열인 경우 사용자가 전환할 수 있는 여러 기본 URL 옵션이 활성화됩니다.
    </ResponseField>

    <ResponseField name="auth" type="Auth">
      <Expandable title="인증 (Auth)">
        <ResponseField name="method" type='"bearer" | "basic" | "key"'>
          모든 API 엔드포인트에 사용되는 인증 전략입니다.
        </ResponseField>
        <ResponseField name="name" type="string">
        API 플레이그라운드에서 사용되는 인증 매개변수의 이름입니다.

        메서드가 `basic`인 경우 형식은 `[usernameName]:[passwordName]`이어야 합니다.
        </ResponseField>
        <ResponseField name="inputPrefix" type="string">
        인증 입력 필드의 접두사로 설계된 기본값입니다.

        예: `inputPrefix`가 `AuthKey`이면 인증 필드의 기본 입력 결과는 `AuthKey`를 상속합니다.
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="playground" type="Playground">
      API 플레이그라운드를 위한 구성
      <Expandable title="플레이그라운드 (Playground)">
        <ResponseField name="mode" default="show" type='"show" | "simple" | "hide"'>
          플레이그라운드를 표시할지, 숨길지, 아니면 사용자 상호작용 없이 엔드포인트만 표시할지(`simple`) 여부입니다.

          더 자세한 내용은 [플레이그라운드 가이드](/api-playground/demo)를 참조하세요.
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="maintainOrder" type="boolean">
      이 플래그를 활성화하면 OpenAPI 페이지의 키 순서가 OpenAPI 파일에 정의된 키 순서와 일치하도록 보장합니다.

      <Warning>이 동작은 곧 기본적으로 활성화될 예정이며, 그 시점에 이 필드는 더 이상 사용되지(Deprecated) 않을 것입니다.</Warning>
    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="openapi" type="string | string[]">
  OpenAPI 파일을 가리키는 URL 또는 상대 경로 문자열 또는 문자열 배열입니다.
  
  예시:
  <CodeGroup>
    ```json 절대 경로
    "openapi": "https://example.com/openapi.json"
    ```
    ```json 상대 경로
    "openapi": "/openapi.json"
    ```
    ```json 다중 경로
    "openapi": ["https://example.com/openapi1.json", "/openapi2.json", "/openapi3.json"]
    ```
  </CodeGroup>

</ResponseField>

<ResponseField name="footerSocials" type="FooterSocials">
  소셜 미디어 계정 객체로, 키:속성 쌍은 소셜 미디어 플랫폼과 계정 URL을 나타냅니다.
  
  예시: 
  ```json
  {
    "x": "https://x.com/mintlify",
    "website": "https://mintlify.com"
  }
  ```
  <Expandable title="하단 소셜 링크 (FooterSocials)">
    <ResponseField name="[key]" type="string">
    `website`, `facebook`, `x`, `discord`, `slack`, `github`, `linkedin`, `instagram`, `hacker-news` 중 하나의 값입니다.
    
    예: `x`
    </ResponseField>
    <ResponseField name="property" type="string">
    소셜 플랫폼으로 연결되는 URL입니다.
    
    예: `https://x.com/mintlify`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="feedback" type="Feedback">
  피드백 버튼을 활성화하기 위한 구성
  <Expandable title="피드백 (Feedback)">
    <ResponseField name="suggestEdit" type="boolean" default="false">
    사용자가 풀 리퀘스트를 통해 편집을 제안할 수 있는 버튼을 활성화합니다.
    </ResponseField>
    <ResponseField name="raiseIssue" type="boolean" default="false">
    사용자가 문서에 대한 이슈를 제기할 수 있는 버튼을 활성화합니다.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="modeToggle" type="ModeToggle">
  다크 모드 토글을 맞춤 설정합니다.
  <Expandable title="모드 토글 (ModeToggle)">
    <ResponseField name="default" type={'"light" or "dark"'}>
      새 사용자에게 항상 라이트 또는 다크 모드를 보여주고 싶을 때 설정합니다. 설정하지 않으면 사용자의 운영 체제와 동일한 모드를 기본값으로 사용합니다.
    </ResponseField>
    <ResponseField name="isHidden" type="boolean" default="false">
      다크/라이트 모드 토글을 숨기려면 true로 설정하세요. `isHidden`과 `default`를 결합하여 문서가 라이트 모드 또는 다크 모드만 사용하도록 강제할 수 있습니다. 예를 들어:
      
      <CodeGroup>
      ```json 다크 모드만 사용
      "modeToggle": {
        "default": "dark",
        "isHidden": true
      }
      ```

      ```json 라이트 모드만 사용
      "modeToggle": {
        "default": "light",
        "isHidden": true
      }
      ```
      </CodeGroup>

    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="backgroundImage" type="string">
  모든 페이지 뒤에 표시될 배경 이미지입니다. [Infisical](https://infisical.com/docs) 및 [FRPC](https://frpc.io)의 예시를 참조하세요.
</ResponseField>
