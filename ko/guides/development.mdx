<Info>
  **사전 요구 사항**:
  - Node.js 버전 19 이상
  - `docs.json` 파일이 포함된 문서 리포지토리
</Info>

사용 중인 운영 체제에서 Mintlify를 설치하고 실행하려면 다음 단계를 따르세요.

<Steps>
<Step title="Mintlify CLI 설치">

```bash
npm i -g mint
```
</Step>

<Step title="로컬에서 미리보기">

`docs.json` 파일이 위치한 문서 디렉토리로 이동하여 다음 명령을 실행합니다:

```bash
mint dev
```

`http://localhost:3000`에서 문서의 로컬 미리보기를 확인할 수 있습니다.

</Step>
</Steps>

## 사용자 지정 포트

기본적으로 Mintlify는 3000번 포트를 사용합니다. `--port` 플래그를 사용하여 Mintlify가 실행되는 포트를 사용자 지정할 수 있습니다. 예를 들어, 3333번 포트에서 Mintlify를 실행하려면 다음 명령을 사용하세요:

```bash
mint dev --port 3333
```

이미 사용 중인 포트에서 Mintlify를 실행하려고 하면 다음 사용 가능한 포트를 자동으로 사용합니다:

```md
Port 3000 is already in use. Trying 3001 instead.
```

## Mintlify 버전

각 CLI 릴리스는 특정 버전의 Mintlify와 연동됩니다. 로컬 미리보기가 프로덕션 버전과 일치하지 않는 경우 CLI를 업데이트해 주세요:

```bash
npm mint update
```

## 링크 유효성 검사

CLI를 사용하여 문서 내의 링크 유효성을 검사할 수 있습니다. 깨진 링크를 식별하려면 다음 명령을 사용하세요:

```bash
mint broken-links
```

## 코드 포맷팅

MDX를 인식하고 형식을 맞추기 위해 IDE 확장 프로그램을 사용하는 것이 좋습니다. VSCode 사용자라면 구문 강조를 위한 [MDX VSCode extension](https://marketplace.visualstudio.com/items?itemName=unifiedjs.vscode-mdx)과 코드 포맷팅을 위한 [Prettier](https://marketplace.visualstudio.com/items?itemName=esbenp.prettier-vscode)를 고려해 보세요.

## 문제 해결

<AccordionGroup>
  <Accordion title='오류: darwin-arm64 런타임을 사용하여 "sharp" 모듈을 로드할 수 없습니다'>

    이는 오래된 노드 버전 때문일 수 있습니다. 다음을 시도해 보세요:
    1. 현재 설치된 CLI 버전 제거: `npm remove -g mint`
    2. Node v19 이상으로 업그레이드.
    3. CLI 재설치: `npm i -g mint`
  </Accordion>

  <Accordion title="문제: 알 수 없는 오류가 발생함">
  
    해결 방법: 장치의 루트로 이동하여 `~/.mintlify` 폴더를 삭제합니다. 그런 다음 `mint dev`를 다시 실행하세요.
  </Accordion>
</AccordionGroup>

최신 CLI 버전에서 무엇이 변경되었는지 궁금하신가요? [CLI 변경 로그](https://www.npmjs.com/package/mintlify?activeTab=versions)를 확인해 보세요.
