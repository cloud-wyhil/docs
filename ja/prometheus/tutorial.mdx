## チュートリアル紹介

このチュートリアルでは、Prometheusの強力なAIエージェント機能を活用して、設計から実行までわずか数行の対話だけで掲示板サービスを構築するプロセスを学習します。開発者は、複雑な文法の悩みなしに、実装したい「意図」を伝えるだけで済みます。

---

## 0. 開始する前に：環境設定 (Setup)

バイブコーディングを開始する前に、JavaベースのSpring Bootプロジェクトがローカル環境でスムーズに動作するように基本的なセットアップが必要です。

### 1. Java 開発環境の設定
コンピュータに以下のツールがインストールされているか確認してください。
* JDK (Java Development Kit) : バージョン17以上のインストールを推奨します。
* Build Tool : Maven または Gradle がインストールされている必要があります。（Prometheusは両方の環境をサポートしています。）

### 2. Spring Initializr の設定 (start.spring.io)
新しいプロジェクトを作成する際、[start.spring.io](https://start.spring.io) で以下のような設定でプロジェクトをダウンロードし、IDEで開いてください。

| 項目 | 設定値 |
| :--- | :--- |
| Project | Maven または Gradle |
| Language | Java |
| Spring Boot | 3.x.x (最新の安定バージョン) |
| Java | 17 または 21 |
| Dependencies | Spring Web, Spring Data JPA, H2 Database, Lombok, Thymeleaf など |

> Tip: 環境設定が難しい場合は、Prometheusに「Java開発環境を確認して、足りない設定を手伝って」と言ってみてください。`run_terminal_command` ツールを通じて現在のシステム状態を点検してくれます。

## 1. 掲示板サーバーの作成 (Backend)

最初に、データの保存とビジネスロジックを担当するサーバー環境を構築します。

### 実行方法
チャットウィンドウに次のように入力して、AIにサーバー開発を指示してください。

> Prompt : "Spring BootとJPAを使って基本的な掲示板サーバーを作って。タイトル、内容、作成者フィールドを持つ Board エンティティが必要だよ。H2データベースを使用して、基本的なCRUDを実行する Service と Controller まで一度に生成して。"

### 動作原理
* Tools 活用 : AIは `create_new_file` ツールを使用して、プロジェクト構造に合ったJavaファイルを生成します。
* Logic Create : ユーザーの要件を分析して、投稿の保存、照会、修正、削除に必要なビジネスロジックを自動的に設計します。

---

## 2. 掲示板画面の作成 (Frontend)

サーバーと連携して、ユーザーが実際に投稿を確認・作成できる画面を構成します。

### 実行方法
サーバーの生成が完了したら、続けてレイアウトの変更および画面の生成を依頼します。

> Prompt : "さっき作ったサーバーと連動する Thymeleaf ベースの HTML 画面を作って。リストページはすっきりとしたテーブル形式がいいし、書き込みボタンを押すと作成フォームが出るようにして。デザインは Bootstrap を使ってモダンな感じでお願い。"

### 動作原理
* Layout Change : AIが `edit_existing_file` を通じて HTML ファイルの構造を作り、Bootstrap クラスを適用して洗練されたレイアウトを完成させます。
* 統合 : 生成されたサーバー API とフロントエンド フォーム (Form) を自動的に接続し、データが正常にやり取りできるようにします。

---

## 3. 実行する (Execution)

作成されたコードをビルドし、実際にサービスが動作するか確認する段階です。

### 実行方法
ターミナルコマンドを直接入力する代わりに、AIに実行を要請してください。

> Prompt : "プロジェクトをビルドしてサーバーを実行して。実行が完了したら、ブラウザでアクセスできるローカルアドレス (URL) を教えて。また、テスト用の投稿を一つ登録する SQL 文も実行して。"

### 動作原理
* System Control : AIが `run_terminal_command` ツールを実行して、ビルドおよび実行コマンドを代わりに遂行します。
* SQL 生成 : AIがデータベース構造に合った SQL クエリを生成して、テストデータを DB に即座に挿入します。

---

## ⚠️ 重要：エラー修正と相互作用 (Iterative Debugging)

LLMが生成するコードは、常に完璧であるとは限らず、環境の違いやバージョンの互換性の問題により、一度に実行されない場合があります。しかし、これは失敗ではなく、バイブコーディングの自然なプロセスです。

* エラーメッセージの伝達 : ビルドエラーやランタイムエラーが発生した場合、ターミナルに表示されたエラー内容をそのままコピーしてAIに伝えてください。
* 相互作用による修正 : 「このエラーがなぜ発生したのか分析して直して」とリクエストすると、AIは `read_file` でコードを再検討し、`edit_existing_file` で修正案を提案します。
* 継続的なフィードバック : 「この部分は私が望む動作ではない」または「別の方法でコードを書いて」といったフィードバックをやり取りしながらコードを完成させていくのがバイブコーディングの核です。

---

## 💡 バイブコーディングのヒント

* 絶え間ない対話 : 「ログイン機能も追加して」、「人気記事のロジックをアップデートして」といった、チュートリアル後の作業も会話を続けるように進めることができます。
* Rules 参照 : プロジェクトの特定のコーディング標準がある場合、AIが Rules を参考にして、チームのスタイルに合ったコードを作成します。
* 安全な実行 : すべてのツールを実行する前に Ask First モードを活用して、AIがどのファイルを修正し、どのようなコマンドを実行するかを検討してください。
